# 一人公司（OPC）工具调研

> 调研时间：2026 年 6 月
> 目的：为 OPC 项目摸清现状——一个人运营公司，市面上已经有哪些工具可用，哪些环节还有空白，作为后续"打造一人公司工具栈"的起点。

---

## 一、背景：什么是 OPC，为什么工具很重要

OPC（One-Person Company / 一人公司）指由单个创始人独立运营、不雇佣全职团队的公司形态。2026 年它已经从"小众实验"变成明确趋势：

- 据多家调研，约 **77% 的一人公司在第一年内实现盈利**，约 1/5 的个体年收入在 10 万~30 万美元之间。
- 驱动力是 **AI + 无代码 + 自动化**：过去需要一个部门完成的事，现在由工具链承担。一份"完整 AI 工具栈"月成本约 **75~200 美元**，可替代相当于每月 600~1000 美元的兼职/外包人力。
- 核心理念的转变：创始人从"亲自做事"变成 **"指挥官"**——把 AI 当作"数字员工"来调度（市场分析、内容、客服、财务各司其职）。

> 对一人公司而言，真正的约束不是"工具能做什么"，而是 **"每个工具占用多少注意力"**。工具越多、整合越差，心智负担越重。这正是 OPC 项目可以切入的痛点。

## 三、国际 vs 中国市场：两套生态

| 维度 | 国际生态 | 中国生态 |
|---|---|---|
| 收款 | Stripe / Lemon Squeezy / Gumroad（MoR 扛全球税务） | 微信/支付宝、国内聚合支付 |
| 记账报税 | Wave / Fondo / QuickBooks | 慧算账、云帐房、好会计、金蝶/用友 |
| 发票 | Invoice 即 PDF，税务自报 | 数电票（全电发票），强监管、需识别专票/普票 |
| 注册合规 | LegalZoom / ZenBusiness，FinCEN BOI | 工商注册 + 代账公司，金税系统 |
| 自动化 | Zapier / Make / n8n | 实在 RPA 等屏幕语义 Agent |

**结论**：海外工具高度模块化、API 友好、可自助拼装；国内财税强合规、偏服务商代办、自动化更依赖 RPA 操作既有软件。一人公司若同时面向中外市场，等于要维护两套栈——**这本身就是 OPC 项目值得抹平的痛点**。

## 四、三大趋势

1. **从"工具"到"数字员工"**：AI Agent 具备目标拆解、自主调度、持续迭代能力，开始承担市场/内容/客服/财务等"岗位"，创始人转为"总指挥"。
2. **从"连接"到"编排"**：自动化从 Zapier 式的 App 连接，走向多 Agent 协同调度（"指挥千军万马"）。
3. **从"多工具"到"一体化 + 低注意力"**：工具数量本身成为负担，Bonsai/HoneyBook/Zoho One 这类整合平台、以及"一个入口管全部"的需求在上升。

## 六、对 OPC 项目的启示（机会与空白）

调研下来，市面工具**单点都很成熟，但"为一个人整体运营公司而设计"的东西很少**。几个潜在切入点：

1. **整合层 / 指挥台**：现有工具各自为政，缺一个"一人公司控制台"统一编排财务、客户、内容、任务，降低注意力切换成本。
2. **中外双栈打通**：同时服务国内外业务的个体，没有人帮他们抹平两套收款/财税/合规体系。
3. **Agent 编排而非又一个 Agent**：市场不缺单个 AI 助手，缺的是把多个"数字员工"按公司职能组织起来的调度框架。
4. **决策与知识沉淀**：邮件/消息管理、日程优先级、决策辅助、个人知识库——这些"一人公司操作系统"层面的东西仍然零散。
5. **合规自动化**：BOI 申报、数电票、年报提醒这类"忘了就罚款"的事，适合做成主动提醒/自动化。
